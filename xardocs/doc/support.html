Getting Support

The developers of CPAINT want your project to be a success in every way. If you run across problems with CPAINT, please feel free to contact us (or other CPAINT developers) through one of the following methods. Remember, it is very helpful if you include the code you are working with (including the CPAINT code to see if there are differences); any debug messages from CPAINT, the backend, or the frontend; or, screenshots if possible. The more information you provide, the better.

» Please do not email the developers individually with your support issues.

» The preferred method for support is to use the CPAINT Discussion Forums.

» If there is no doubt in your mind that you have found a defect in any of the CPAINT code, please enter a defect report in the SourceForge CPAINT Bug Tracking System†.

» If you have a suggestion for us, feel free to enter it in the SourceForge CPAINT Feature Request System†. We will review requests and may incorporate your suggestions into future versions of CPAINT.

† You will need a user account on SourceForge to utilize these systems. Only the developers can view tickets and respond to them. Requests for development assistance outside of CPAINT will be removed.
