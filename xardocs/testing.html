3. Dynamic Data Properties

3.1
 File Upload (hooked)

Input shows upload sources.

Output shows MIME icons with link to uploaded files (if any).

File is uploaded via DD property and copied to the server's filesystem.

File is uploaded via DD property and the properties value is a semicolon delimited string of file ids.

Validation field accepts semicolon delimeted keywords.

Validation of 'single' means only one file is allowed to be selected for the instance.

Methods of upload can be provided in validation field:
  • (+/-)external - ability to use external sources (http/ftp)
  • (+/-)upload    - ability to upload
  • (+/-)trusted  - ability to select trusted files
  • (+/-)stored   - ability to select previously stored files
3.2
 Text Upload (hooked)

Input shows a text area + upload sources.

Adding an filemanager or selecting uploaded files, and adds a "magic link" #ulid:NN# to the text area.

Text output shows the text - filemanager transform hooks replaces the magic links with icons and link to uploaded files.
