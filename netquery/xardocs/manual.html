Netquery Administration

Netquery provides users of your site with a range of common network tools, including Whois lookups, DNS queries, ICMP ping, traceroute and server port checking. Each feature can be enabled or disabled individually by the administrator. The administrator can also configure the Whois feature to specify which Whois server is associated with each domain name.ext entry and can add, modify and delete those entries in the Netquery Whois data table.

If you are using this Netquery module on a Microsoft Windows server, be sure to check that option as noted below. Other options determine which Netquery functions are available to users of your web site. You should select at least one, or else uninstall this module completely. Please note that the local system execution option requires read/execute permission for your operating system's ping and traceroute executables. If you check the Capture Logfile option, results will be stored in the capturedata.txt file in the Netquery module directory. This capturedata.txt file must have write permission for the server.
Enable/Disable Features
Server Is MS Windows Check this if you are using a Windows server. (Affects local ping and traceroute exec.)
Enable Local Execution Ping and traceroute functions use local system calls. (Requires execute permission.)
Enable Whois Search Whois lookups are automated based on server->domain data entries. (See below.)
Enable Whois IP Search Whois lookups for IP addresses using IANA's query server.
Enable DNS Lookup Domain name service forward and reverse lookups.
Enable DNS (Dig) Query DNS queries using the *nix dig function. (Not available on Windows servers.)
Enable ICMP Ping Ping is limited to a maximum of 10 counts. (Local excution if checked above.)
Enable Traceroute Traceroute from server to specified target. (Local execution if checked above.)
Enable Port Check Check specified server for availability of specified port.
Enable Capture Log Logs all Netquery queries in a capturedata.txt file on your server. (Requires write permission.)
Edit-Add-Delete Whois Lookup Links
General This admin menu option displays a list of whois server - domain.ext pairs.
Edit This option for each listed whois pair allows you to edit that "linkage".
Delete This option for each listed whois pair allows you to remove that "linkage".
Add New This option allows you to add a new whois server - domain.ext pair.
*NOTE* Deleting all whois pairs from the Netquery database will produce an error. Disable the whois feature in the main admin menu instead.
