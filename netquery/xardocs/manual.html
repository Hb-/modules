Netquery Administration

Netquery provides users of your site with a range of common network tools, including Whois lookups, DNS queries, ICMP ping, traceroute and server port checking. Each feature can be enabled or disabled individually by the administrator. Local execution and/or a remote server script can be used for the ping and traceroute features. The administrator can also configure the Whois feature to specify which Whois server is associated with each domain name.ext (TLD) entry and can add, modify and delete those entries in the Netquery Whois data table.

Please Note: The local ping and traceroute options require read/execute permission for your operating system's ping and traceroute executables. The Netquery Log option requires write permission for the specified path/filename. (Installation default is var/logs/netquery.log)
Netquery Configuration Options
Enable Whois Search Whois lookups are automated based on server->domain data entries. (See below.)
Enable Whois IP Search Whois lookups for IP addresses using IANA's query server.
Enable DNS Lookup Domain name service forward and reverse lookups.
Enable DNS (Dig) Query DNS queries using the *nix dig function. (Not available on Windows servers.)
Enable Local ICMP Ping Ping is limited to a maximum of 10 counts. (Excutable entry may include full path.)
Enable Remote ICMP Ping Ping count is determined by remote script. (Remote script url?param=target)
Enable Local Traceroute Traceroute from your server to specified target. (Excutable entry may include full path.)
Enable Remote Traceroute Traceroute from remote server to specified target. (Remote script url?param=target)
Enable Port Check Checks specified server for availability of specified port.
Enable Netquery Log Logs all Netquery actions to the specified path/filename. (Requires write permission.)
Edit-Add-Delete Whois Lookup Links
General This admin menu option displays a list of whois server - TLD pairs.
Edit This option for each listed whois pair allows you to edit that "linkage".
Delete This option for each listed whois pair allows you to remove that "linkage".
Add New This option allows you to add a new whois server - TLD pair.
*NOTE* Deleting all whois pairs from the Netquery database will produce an error. Disable the whois feature in the main admin menu instead.
