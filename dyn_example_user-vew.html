Reference a dataobject by name in 
<xar:data-view />

This approach involves the least amount of code in user-view.xt.

Tag in user-view.xt: 

<xar:data-view object="dyn_example" tplmodule="dyn_example" />

Template invoked by the tag:

Pass values to 
<xar:data-view />

Return an array from your module function and the values will be available in your template.

In the case of this page, the module function is in dyn_example/xaruser/view.php . It returns $data. Thus, if you set a value for $data['something'] in the module function, you can use $something in the template to output that value.

Have a look in view.php, and you'll see that our $data array supplies the values for variables we use in user-view.xt, specifically $mylist, $items_per_page and $startnum.

Tag in user-view.xt: 

    <xar:data-view object="$mylist" numitems="$items_per_page"
        startnum="$startnum" tplmodule="dyn_example" />

Template invoked by the tag:

modules/dyn_example/xartemplates/objects/showview-dyn_example.xt

Get items in the template with 
<xar:data-getitems />

As in Approach #2, we have set some variables in view.php for use in user-view.xt. Those variables serve as arguments in 
<xar:data-getitems/>

Tag in user-view.xt:

In user-view.xt, we retrieve items with... 

<xar:data-getitems object="$mylist" values="$values" properties="$properties"
    numitems="$items_per_page" startnum="$startnum" />
...and then iterate over the items with 
<xar:foreach />
.
